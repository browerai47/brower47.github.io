public class ReceiptPrinter {
    private final double taxRate = 0.08;
    private double subtotal = 0.0;
    double tax = 0.0;
    double finalTotal = 0.0;
    private Maker hamburger;
    private Maker hotdog;
    private Maker soda;
    private Maker coffee;
    public  ReceiptPrinter(Maker hamburger, Maker hotdog, Maker soda, Maker coffee){
        this.hamburger = hamburger;
        this.hotdog = hotdog;
        this.soda = soda;
        this.coffee = coffee;
        calculateTotals();
    }
    
    private void calculateTotals() {
      // Logic would calculate totals here using the Maker instances
      this.subtotal = (hamburger.quantity * hamburger.unitPrice) + /* ... etc ... */;
      this.tax = this.subtotal * this.taxRate;
      this.finalTotal = this.subtotal + this.tax;
    }

    // A print method might format the output to console
    public void printReceipt() {
        System.out.println("*  Subtotal: " + subtotal + "                                     *");
    }
    
    @Override
    public String toString() {
      // Logic to format final receipt string for System.out.println(print1)
      return "*  Final Total: " + finalTotal + "                                    *"; 
    }
}

class Maker {
    private String itemName;
    private double unitPrice;
    private int quantity;

    public Maker(String itemName, double unitPrice, int quantity) {
        this.itemName = itemName;
        this.unitPrice = unitPrice;
        this.quantity = quantity;
    }
    
    // Getters and setters omitted for brevity
    
    @Override
    public String toString() {
      return "*  Item: " + this.itemName + ", Qty: " + this.quantity + "                                    *";
    }
}

Project Logic Explanation

The Java program uses standard console I/O (`java.util.Scanner`) to prompt a user for quantities of four menu items. It then calculates and prints a formatted receipt including itemized costs, subtotals, tax (8%), and a final total.

  • The `Receipt` class contains the `main` method which drives user interaction.
  • The `Maker` class acts as a data structure to hold the item name, price, and quantity.
  • The `ReceiptPrinter` class handles the calculation logic and formatting the final output string using its `toString()` override.
